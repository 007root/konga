A comma-separated list of URIs prefixes that point to your Route. For example: /my-path. At least one of hosts, uris, or methods should be specified.

A comma-separated list of HTTP methods that point to your Route. For example: GET,POST. At least one of hosts, uris, or methods should be specified.

When matching a Path via one of the paths prefixes, strip that matching prefix from the upstream URI to be requested. Default: true.

When matching a Route via one of the hosts domain names, make sure the request Host header is forwarded to the upstream service. By default, this is false, and the upstream Host header will be extracted from the configured upstream_url.

A comma-separated list of Protocols that point to your Route. For example: http,https. At least one of http or https should be specified.
